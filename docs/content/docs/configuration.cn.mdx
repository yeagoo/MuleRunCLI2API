| 环境变量 | 默认 | 说明 |
|----------|------|------|
| `CLI2API_PORT` | `51222` | HTTP 监听端口 |
| `CLI2API_API_KEYS` | *(空)* | 入站 API key 白名单（逗号分隔）；空 = 不鉴权 |
| `CLI2API_JOBSTORE_DSN` | *(空 = 内存)* | libsql DSN（`file:...` / `libsql://...?authToken=...`） |
| `CLI2API_JOB_RETENTION` | `168h` | job 行保留时长；`0s` = 不过期（reaper 跳过 sweep） |
| `CLI2API_REAPER_INTERVAL` | `1h` | reaper 扫描周期；`0s` = 禁用 |
| `CLI2API_JOB_HARD_CAP_MULT` | `3.0` | in-flight job 硬上限倍数（`expires_at + retention × N` 后强删）。范围 1.0–1000.0 |
| `CLI2API_IMAGE_TIMEOUT` | `5m` | 同步图像/语音最长等待 |
| `CLI2API_POLL_INTERVAL` | `2s` | 上游轮询初始周期 |
| `CLI2API_POLL_MAX_INTERVAL` | `10s` | 轮询退避上限 |
| `CLI2API_USAGE_DSN` | *(空 = 内存)* | 用量记录的 libsql DSN —— 见[用量端点](/cn/docs/endpoints/usage)。 |
| `CLI2API_USAGE_RETENTION` | `720h`（30 天） | 比此旧的记录会被 reaper 删掉。`0s` = 不过期。 |
| `CLI2API_USAGE_BUFFER` | `1024` | 异步 writer 通道大小。 |
| `CLI2API_LOG_LEVEL` | `info` | `debug` / `info` / `warn` / `error` |
| `MULERUN_TOKEN` | *(读 cache 文件再 `~/.mulerun/`)* | 上游 MuleRun token（muk- 或 JWT） |
| `MULERUN_API_BASE_URL` | `https://api.mulerun.com` | 上游网关 base URL |
| `CLI2API_TOKEN_CACHE` | *(`~/.config/mulerun/oauth_cache.json`)* | OAuth cache 路径。生产容器/独立用户 systemd 部署里指向可写文件，cli2api 会 refresh JWT 并把兑换出的 muk- 缓存进来。 |
| `CLI2API_OAUTH_TIMEOUT` | `10s` | JWT 兑换 + token refresh 的单次调用超时。控制冷启动开销，避免 k8s/PaaS readiness probe 误判。 |
| `MULERUN_OAUTH_BASE_URL` | `https://mulerun.com` | OAuth/平台 host（cli-token + oauth2/token 端点），跟上面的推理网关是不同的。 |

仓库里的 `.env.example` 有可复制的模板。

## 凭证发现顺序

1. `MULERUN_TOKEN` 环境变量
   - `muk-...` → 直接用
   - `eyJ...`（JWT）→ 启动时兑换，**不**写回磁盘（见下方提示）
2. `CLI2API_TOKEN_CACHE` 如果设了，否则 `~/.config/mulerun/oauth_cache.json`
   - 有 `cli2api_muk_key` → 快路径，没有 HTTP
   - 否则读 `access_token` + `refresh_token`：过期就 refresh、兑换出 muk-、
     把 muk- 写回文件给下次启动用
3. `~/.mulerun/{auth,credentials,token}.json`（旧版本）

启动日志的 `token_source` 字段显示用了哪个来源。

**env-JWT 限制**：通过 `MULERUN_TOKEN` 传的 JWT 每次重启都会重新兑换（我们不写回
env）。要享受磁盘缓存的话，把 JWT 放进 `CLI2API_TOKEN_CACHE` 文件里。

## Reaper 保留期语义

- **终态 job**（`completed` / `failed`）：`expires_at` 一过就删
- **in-flight job**（`queued` / `in_progress`）：保留到
  `expires_at + retention × CLI2API_JOB_HARD_CAP_MULT`（默认 3×）—— 给客户端
  足够时间继续轮询
- `CLI2API_JOB_RETENTION=0` 整体禁用 sweep —— 任何 job 都不会被自动删
