## 怎么拿你的 `muk-` API key

cli2api 需要在 `MULERUN_TOKEN` 里放一个 **`muk-` API key**——绑账号的长期稳定 key，
MuleRun API 网关认这个。`mulerun login` 拿到的是**会话 JWT**，**不能用**（见下）。

`mulerun` CLI 每次 `studio` 调用时内部把登录态换成 `muk-` key，但不直接打印。
抓你自己的：

```sh
# 包一层 mulerouter 打印 CLI 传给它的 key，跑一次,再还原。
# （进阶——目前没有官方的"打印 key"命令。）
W=$(readlink -f ~/.mulerun/vendor/mulerouter/node_modules/.bin/mulerouter)
mv "$W" "$W.real"
printf '#!/bin/bash\nenv | grep ^MULEROUTER_API_KEY= | cut -d= -f2\n' > "$W"
chmod +x "$W"
mulerun studio list >/dev/null 2>&1   # 打印 muk-... 到 stdout
mv "$W.real" "$W"
```

然后 `export MULERUN_TOKEN=muk-...`。这个 key 跨次调用稳定，只需抓一次
（存进 `.env` / systemd 单元）。

## 切换 MuleRun 账号（或轮换 key）

`muk-` key 绑一个账号。要让 cli2api 指向另一个账号（比如有积分的那个），
重新登录、抓新 key、更新 env。

```sh
# 1. 登录你要用的账号（浏览器 OAuth）。
mulerun logout
mulerun login
mulerun user balance        # 确认新账号 balance > 0

# 2. 抓新账号的 muk- key。
W=$(readlink -f ~/.mulerun/vendor/mulerouter/node_modules/.bin/mulerouter)
mv "$W" "$W.real"
printf '#!/bin/bash\nenv | grep ^MULEROUTER_API_KEY= | cut -d= -f2\n' > "$W"
chmod +x "$W"
NEWMUK=$(mulerun studio list 2>/dev/null | grep '^muk-' | head -1)
mv "$W.real" "$W"

# 3. 原地更新 MULERUN_TOKEN（保留其它配置），然后重启。
sudo sed -i "s|^MULERUN_TOKEN=.*|MULERUN_TOKEN=$NEWMUK|" /etc/cli2api/env
sudo systemctl restart cli2api

# 4. 验证 —— token_source 应为 env、无 JWT 警告、调用成功。
journalctl -u cli2api -n2 --no-pager
```

如果你已经有新账号的 `muk-` key，跳过 1–2 步，只用它跑第 3 步。直接前台跑
（非 systemd）的话，把改 env 换成 `export MULERUN_TOKEN=muk-...` 再重启进程。

## 常见错误

### `no mulerun credentials found`
`export MULERUN_TOKEN=muk-...`（见上）。启动日志的 `token_source` 字段显示读了哪个
文件/环境变量。

### 任意端点 `502 upstream HTTP 401` / `Invalid API Key format`
你用的是 OAuth JWT，不是 `muk-` key。`mulerun login` 缓存的会话 JWT 被网关在
**所有**端点（图像/视频/音频和 chat）拒绝。cli2api 检测到 JWT 会在启动时警告。
修复：把 `MULERUN_TOKEN` 设成你的 `muk-` key（见上面"怎么拿 muk- key"）。

### `502 upstream HTTP 402` —— MuleRun 额度耗尽
`402 Payment Required` 是你的 MuleRun **账户余额用光了**，不是限流（那是 429），
也不是 cli2api 的 bug —— 网关只是把上游的 402 透传回来。查余额：

```sh
mulerun user balance      # data.balance —— 接近 0 就是没钱了
mulerun user wallets      # daily / monthly / longterm credits 细分
```

去 [mulerun.com](https://mulerun.com) 充值（你登录的那个账号），然后重试即可，
不用重启。快耗尽时的典型现象：

- **premium 模型先挂**（`gpt-image-2`、`nano-banana*`、`midjourney` 单价高），
  而最便宜的 `wan2.6-t2i` 还能再出几张，然后也 402。
- 额度紧张时优先 `wan2.6-t2i`；`gpt-image-2` / `midjourney` 留给真正需要的图。

### `/v1/chat/completions` 返回 `Model 'gpt-5' is not supported`
认证通过了，但 studio 平面的 `muk-` key 不暴露 chat 模型。文本端点
（`/v1/chat/completions`、`/v1/messages`、`/v1/responses`）需要 MuleRun 单独的
LLM gateway key；`muk-` studio key 只覆盖图像/视频/音频。

### `vendor_error: code 3005 / 3006 / ...`
这是真实的上游错误（MiniMax/Seedance/Wan 服务自身挂了）。cli2api 工作正常——它把
结构化的上游错误传给客户端。重试或换模型。

### `404 unknown image model: dall-e-3`
cli2api **不**把 OpenAI 名映射到 MuleRun。请用真名（`gpt-image-2` / `wan2.6-t2i`
/ `midjourney`）。跑 `curl localhost:51222/v1/models | jq '.data[].id'` 看完整列表。

### 视频 / 音乐 job 卡在 `queued`
检查 `CLI2API_JOB_RETENTION` 和 `CLI2API_JOB_HARD_CAP_MULT`：retention 太短 +
multiplier 太小，reaper 会在轮询完成前删掉 job。默认（7d / 3×）够用，只在短期测试
时才调小。

### 重启后 job ID 找不到了
内存 store 重启即丢。用 `CLI2API_JOBSTORE_DSN=file:...` 或远端 libsql 持久化。

### `request body too large` (400)
单次请求上限 64 MB。这是 chi 的 `RequestSize` 中间件，包装成 OpenAI 风格 400。

### SSE 流式响应不出增量
反向代理没关 `proxy_buffering`——nginx / Caddy 都要单独配。另外确认客户端用
`stream=True`，不是 `json()` 整收。

## FAQ

### 为什么做代理而不直接调 MuleRun？
`/v1/chat/completions` 和 `/v1/messages` 本来就兼容，想直接调也行。但图像/视频/音频
走 `/vendors/{vendor}/...` 异步 job 形式，跟 OpenAI 完全不同。cli2api 把这层隐藏掉，
让现有 SDK 代码不改就能跑。

### 会缓存结果吗？
不会。每次调用都是新的 MuleRun task。要缓存在应用层或 CDN 层做。

### 能跑在 Lambda / Cloud Function 里吗？
文本和图像同步端点可以。视频/音乐显式异步——客户端轮询，实例随便重启。配合 libsql
持久化 store 即可。

### 多租户怎么办？
没有内置用户/配额。`CLI2API_API_KEYS` 是平面 allow-list。要做配额请放在 API 网关后面。
