## Credential discovery order

1. `MULERUN_TOKEN` env var

2. `CLI2API_TOKEN_CACHE` if set, else `~/.config/mulerun/oauth_cache.json`
   (mulerun-cli â‰Ą0.1.0)
   - `cli2api_muk_key` field â†’ fast-path, no HTTP
   - else `access_token` + `refresh_token` â†’ refresh if expired, exchange to
     muk-, persist muk- back for the next startup
3. `~/.mulerun/{auth,credentials,token}.json` (older versions)

The startup log's `token_source` field shows which source was used.

**Env-JWT caveat**: a JWT passed via `MULERUN_TOKEN` is re-exchanged on every
restart (we don't write back to env). To benefit from disk caching, put your
JWT in a file at `CLI2API_TOKEN_CACHE` instead.
