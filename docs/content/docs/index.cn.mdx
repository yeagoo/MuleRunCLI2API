**cli2api** 把 [MuleRun](https://mulerun.com) 的生文、生图、生视频、TTS、音乐能力，
封装成 **OpenAI 和 Anthropic 兼容**的 HTTP 服务。现有的 `openai` / `anthropic` SDK
改一个 `base_url` 就能跑——无需改业务代码。

单二进制 Go 服务（~12 MB），distroless 镜像可用，零 CGO 依赖。

  <Card title="文本" href="/cn/docs/endpoints/chat">

## 为什么要做代理？

MuleRun 的 `/v1/chat/completions` 和 `/v1/messages` 本来就 OpenAI/Anthropic 兼容，
直接调没问题。但图像/视频/音频走 `/vendors/{vendor}/...` 的**异步 job** 形式，
request shape 跟 OpenAI 完全不同。cli2api 把这层差异隐藏掉，让现有 SDK 代码不改就能跑：

- **同步**的图像和语音端点，内部轮询 MuleRun 直到完成
- **异步**的视频和音乐端点，暴露干净的提交 + 轮询 job API
- **70+ 模型**已注册：`gpt-image-2`、`nano-banana` 全家、`sora-2`、`veo`、
  `kling-v3-omni`、`wan2.6-*`、`midjourney`、`seedance-2.0-*`、`speech-2.8-hd`、
  `music-2.5` 等等
